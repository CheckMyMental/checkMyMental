# 치트키 정리

## 1. 1단계 Intake 치트키

- **우울증패스**
  - 위치: 일반 대화 입력창 (1단계 진행 중일 때)
  - 효과:
    - Intake 질문 과정을 건너뛰고, 미리 정의된 `Summary String`을 `intake_summary_report`에 바로 주입.
    - 이후 2단계 Hypothesis(RAG 가설 검색)부터 바로 진행.
  - 내부에서 사용하는 고정 Summary String 예시:
    - 주요_증상: 극심한 무기력감과 우울한 기분이 거의 매일 지속됨.
    - 수면: 깊게 잠들지 못하고 자주 깨며, 새벽에 일찍 깸.
    - 식욕_체중: 식욕 저하로 최근 체중이 급격히 감소함.
    - 활력_에너지: 에너지가 거의 없고, 일상적인 일에도 심한 피로감을 느낌.
    - 신체증상: 가슴 답답함, 두통 및 만성적인 통증 동반.

## 2. 3단계 Validation 치트키

Validation 단계에서 **질문 리스트가 이미 생성된 이후**에 사용.

- **개발자 상**
  - 효과: 모든 Validation 질문의 응답을 **5점(매우 자주/항상)** 으로 채운 뒤, 곧바로 검증 결과 계산으로 이동.
  - 내부 동작:
    - `validation_answers = [5, 5, ..., 5]` (질문 수만큼)
    - `validation_current_index` 를 마지막으로 이동 → 평가용 LLM 호출.

- **개발자 하**
  - 효과: 모든 Validation 질문의 응답을 **1점(전혀 그렇지 않다)** 로 채운 뒤, 곧바로 검증 결과 계산으로 이동.
  - 내부 동작:
    - `validation_answers = [1, 1, ..., 1]`
    - `validation_current_index` 를 마지막으로 이동 → 평가용 LLM 호출.

※ 주의: 치트키는 **Validation 질문이 한 번 이상 생성된 상태**에서만 동작함.

---

## 3. 4단계 Severity(심각도 평가) 치트키

Severity 단계에서 **심각도 질문 리스트가 이미 생성된 이후**에 사용.

- **개발자 상**
  - 효과: 모든 심각도 평가 질문의 응답을 **5점** 으로 채운 뒤, 바로 최종 심각도 평가 LLM을 호출.
  - 내부 동작:
    - `severity_answers = [5, 5, ..., 5]`
    - `severity_current_index` = 전체 문항 수 → 최종 평가로 직행.

- **개발자 하**
  - 효과: 모든 심각도 평가 질문의 응답을 **1점** 으로 채운 뒤, 바로 최종 심각도 평가 LLM을 호출.
  - 내부 동작:
    - `severity_answers = [1, 1, ..., 1]`
    - `severity_current_index` = 전체 문항 수 → 최종 평가로 직행.

---

## 사용 팁

- 개발/디버깅 용도로만 사용하고, 실제 사용자 세션에서는 숨기거나 비활성화하는 것이 안전합니다.
- 한 세션 안에서 단계별 치트키를 조합해서 빠르게 **1→2→3→4→5단계 전체 플로우**를 재현할 수 있습니다.
- 치트키 입력 후에는 터미널 로그(Validation/Severity/Solution Debug)를 함께 확인하면 흐름을 추적하기 쉽습니다.
