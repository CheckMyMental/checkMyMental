# Stage 3: Validation (진단 검증)

## System Instruction
- 입력된 '가설 리포트' 문자열을 분석한다.
- 리포트의 감별점(예: `*다음 단계에서...`)을 기반으로, Top 후보군을 감별할 수 있는 사용자 질문(선택형)을 1~2개 생성한다. (User Step 4 전반부)
- 사용자의 답변을 입력받는다.
- 사용자 답변을 근거로 Top 3 중 가장 가능성 높은 단일 질환명을 선택(Scoring)한다. (User Step 4 후반부)
- 선택된 단일 질환명을 문자열로 출력한다.
- [중요 규칙] 최종 출력하는 질환명은 반드시 Stage 2의 Top 3 후보 질환명 중 하나를
  '철자, 공백, 괄호, 표기'까지 포함해 "정확히 동일한 문자열"로 사용한다. 동의어/약어/다른 표기는 금지한다.

## Required Context
- (없음 - 이전 단계의 문자열에 의존)

## Input Prefix
Hypothesis String:

## Output Prefix
Validated String:

## Prompt

입력된 'Hypothesis String'을 분석하라.

1. 리포트 하단의 '감별할 것' 내용을 바탕으로, 후보 질환들을 감별할 수 있는 명확한 **선택형 질문**을 1개 생성하여 사용자에게 제시한다.
2. (사용자 답변 대기)
3. 사용자의 답변을 입력받는다.
4. 사용자의 답변을 근거로, 리포트의 Top 3 후보 중 가장 적합한 단일 질환명을 **확정**한다.
5. 확정된 질환명만 'Validated String'으로 출력한다.
   - 이때 질환명은 Stage 2의 후보 이름 중 하나를 "그대로" 사용한다. (철자/공백/괄호 포함)

## Output Format (Example)
```
Validated String:
[확정된 단일 질환명]
```

## Strict Output Rules
- 출력에는 추가 설명/점수/근거를 포함하지 않는다.
- 출력은 정확히 위의 형식만 사용한다.

## Few-shot

Input (Stage 2의 Output):
```
Hypothesis String:
가설 리포트:
1. 주요 우울 장애 (확률: 높음): 핵심 기준은 '거의 매일 지속되는 우울한 기분 또는 흥미/즐거움의 현저한 저하'입니다.
2. 번아웃 (직무 소진) (확률: 중간): 핵심 기준은 '주로 직무 및 업무 환경과 관련된 정서적 고갈'입니다.
3. 범불안 장애 (확률: 낮음): 핵심 기준은 '통제하기 어려운 과도한 불안과 걱정'입니다.
*다음 단계에서 '주요 우울 장애'와 '번아웃'의 '전반적인 흥미 저하' vs '직무 관련 흥미 저하'를 감별할 것.
```

(LLM이 생성하는 질문):
```
A: 현재 느끼시는 무기력감이나 흥미 저하가, 다음 중 어떤 것에 더 가깝나요?

(A) 주로 회사 업무나 일과 관련해서만 그렇게 느껴집니다.
(B) 주말이나, 예전에 좋아했던 취미 활동 등 삶의 모든 면에서 전반적으로 그렇게 느껴집니다.
```

(사용자 답변):
```
User: (B)에 더 가까운 것 같아요.
```

Output (LLM의 최종 선택 문자열):
```
Validated String:
주요 우울 장애
```
